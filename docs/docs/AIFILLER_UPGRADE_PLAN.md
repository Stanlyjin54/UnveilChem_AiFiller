# Aifiller 升级执行方案

## 1. 项目概述

### 1.1 项目背景
Aifiller 是一款化学文档参数提取工具，当前已具备基础的文档解析功能。为提升解析能力和用户体验，计划整合高级 OCR 工具（Pix2Text 和 PaddleOCR），并采用分版本策略，为不同层级用户提供差异化服务。

### 1.2 升级目标
- 整合 Pix2Text 和 PaddleOCR 到现有解析器架构
- 实现分版本策略（基础版/专业版/企业版）
- 提升文档解析准确率和支持格式
- 优化用户体验和系统性能

## 2. 技术架构

### 2.1 现有架构
- **后端**：FastAPI + SQLAlchemy ORM
- **前端**：React
- **解析器**：基于 BaseDocumentParser 抽象类的解析器架构
- **数据库**：SQLite/MySQL

### 2.2 升级架构
- **新增 OCR 引擎**：Pix2Text（复杂 PDF 解析）、PaddleOCR（高精度 OCR）
- **分版本策略**：基础版/专业版/企业版
- **资源级访问控制**：基于用户版本的解析器访问权限

## 3. 升级内容

### 3.1 解析器整合

#### 3.1.1 整合 Pix2Text
- **功能**：复杂 PDF 解析，支持表格、公式、图片提取
- **资源级别**：medium（专业版/企业版可用）
- **实现文件**：`backend/app/services/document_parsers/pdf_parser.py`

#### 3.1.2 整合 PaddleOCR
- **功能**：高精度 OCR 识别，支持多语言
- **资源级别**：medium（专业版/企业版可用）
- **实现文件**：`backend/app/services/document_parsers/image_parser.py`

#### 3.1.3 整合 parsers 目录高级解析器
- **功能**：统一解析器，整合 Apache Tika、Camelot-py、Tesseract OCR、PyMuPDF 等
- **资源级别**：high（仅企业版可用）
- **实现文件**：`backend/app/services/document_parsers/advanced_parser.py`

### 3.2 分版本策略实现

#### 3.2.1 用户模型扩展
- **新增字段**：
  - `version`：用户版本（basic/pro/enterprise）
  - `monthly_quota`：月使用配额
  - `used_quota`：已使用配额
  - `last_reset`：配额重置时间
- **实现文件**：`backend/app/models/user.py`

#### 3.2.2 解析器资源级别
- **资源级别定义**：
  - `low`：基础资源，所有版本可用
  - `medium`：中级资源，专业版/企业版可用
  - `high`：高级资源，仅企业版可用
- **实现文件**：`backend/app/services/document_parsers/__init__.py`

#### 3.2.3 解析器管理器升级
- **新增功能**：
  - `get_available_parsers`：根据用户版本返回可用解析器
  - 版本化解析器选择逻辑
  - 配额管理
- **实现文件**：`backend/app/services/document_parsers/parser_manager.py`

### 3.3 版本功能对比

| 功能 | 基础版 | 专业版 | 企业版 |
|------|--------|--------|--------|
| 基础 PDF 解析 | ✅ | ✅ | ✅ |
| 基础 OCR 识别 | ✅ | ✅ | ✅ |
| Pix2Text 复杂 PDF 解析 | ❌ | ✅ | ✅ |
| PaddleOCR 高精度识别 | ❌ | ✅ | ✅ |
| 高级统一解析器 | ❌ | ❌ | ✅ |
| 月使用配额 | 100 | 500 | 无限 |
| 优先处理 | ❌ | ❌ | ✅ |

## 4. 执行步骤

### 4.1 环境准备
1. **创建虚拟环境**：
   ```bash
   python -m venv venv
   venv\Scripts\activate  # Windows
   source venv/bin/activate  # Linux/Mac
   ```

2. **安装依赖**：
   ```bash
   pip install -r requirements.txt
   pip install pix2text paddleocr
   ```

- 实现 `image_parser.py` 的 PaddleOCR 整合

#### 4.2.3 升级解析器管理器

## 5. 部署方案

### 5.1 开发环境
- **后端**：FastAPI 开发服务器

- **数据库**：SQLite

- **容器化**：Docker + Docker Compose

## 6. 风险评估

### 6.1 技术风险
- **依赖冲突**：Pix2Text 和 PaddleOCR 可能与现有依赖冲突
  - **缓解措施**：使用虚拟环境隔离依赖，测试兼容性

- **解析器性能**：高级解析器可能消耗大量资源
  - **缓解措施**：实现资源监控和自动扩展，优化解析器选择逻辑

### 6.2 业务风险
- **用户体验**：版本差异可能导致用户困惑
  - **缓解措施**：清晰的功能对比和升级引导

- **配额管理**：配额不足可能影响用户体验
  - **缓解措施**：提前通知用户，提供灵活的配额购买选项
